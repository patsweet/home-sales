Analysis of New Castle County property data by subdivision¶

In this excercise, I take subdivision level property data and try to answer the following questions:

  1. In which subdivisions are houses selling the fastest? Selling slowest?
  2. In which subdivisions is the median sale price improving most? Dropping most?
  3. Where can a seller get the best deal in terms of sale price / original listing price?

Methodology¶

To tackle these questions, I'm using Pandas - a Python library for data analysis - and SciPy - a mathematics, science and engineering library - to filter, group and ultimately run a ordinary least squares linear regression to determine the positive or negative movement on a number of variables.

About the data¶

The subdivision level data comes from Trend MLS, a real estate data company that provides research for realtors in New Castle and Kent counties. Trend was only able to provide us with subdivision- and Zip code-level data for 2010-2013, inclusive. Because of the large size of Zip codes, we'll focus exclusively on subdivisions. The basic structure of the .csv data is:

  • Subdivision
    • The name of the subdivision
  • Year
    • The data year
  • Sales
    • Number of transactions
    • (Note this doesn't necessarily mean number of homes)
  • Median Sale Price
    • Median price of transactions
  • Avg. Sale Price
    • Average price of transactions
  • Median Days on Market
    • Median number of days the house was listed for sale.
  • Avg. Days on Market
    • Average number of days the house was listed for sale.
  • Median Sale Price / Original Listing Price
    • The ratio between sale price and the original listing price
    • A value above 1.0 means the homes sold for more than the original listing price.
  • Median Sale Price / Listing Price
    • The ratio between the sale price and last listing price.
    • Again, a value above 1.0 means the homes sold for more than the listing price.

Known Issues¶

There are two issues that directly affect this analysis:

  1. Not all subdivisions have data for every year.
    • This could simply be missing data or zero (0) homes were sold in that subdivision for that year.
    • WORKAROUND: In this excercise, we are ommitting all subdivisions that don't have data for all four years. This reduces our dataset from roughly 1,400 subdivisions to less than 700.
  2. In many subdivisions, there is a very small number of sales.
    • This means we are analyzing medians and trying to find trends in some areas where the number of transactions is very small.
    • WORKAROUND: At this point we are not controlling for the number of sales. We'll have to investigate whether to ommit data that don't meet a threshold for the number of sales. In combination with issue No. 1, though, we risk severly diluting our dataset.

def calculateSlope(df, attribute, r_squared_threshold=R2_THRESHOLD):
    """Gets the slope of an attribute of the dataframe along with
    min, max and r-squared values."""
    # Using scipy's linregress, get the slope of dom/year for each subdivision
    slope = pd.DataFrame(df.apply(lambda x: linregress(y=x[attribute], x=x.Year)[0]))
    r2 = pd.DataFrame(df.apply(lambda x: linregress(y=x[attribute], x=x.Year)[2]**2))
    output = pd.merge(left=slope, right=r2, left_index=True, right_index=True)
    
    # Get the max and min DOM for each subdivision.
    output = pd.merge(left=output, right=pd.DataFrame(df[attribute].min()), left_index=True, right_index=True)
    output = pd.merge(left=output, right=pd.DataFrame(df[attribute].max()), left_index=True, right_index=True)
    output= pd.merge(left=output, right=pd.DataFrame(df['Sales'].sum()), left_index=True, right_index=True)
    
    # Fix Column names
    attribute = re.sub(r'[\W]+', '_', attribute).lower()
    output.columns = [attribute+'_slope', attribute+'_r2', attribute+'_min', attribute+'_max', 'sales_count']
    
    return output, output[output[attribute+'_r2'] >= r_squared_threshold]
